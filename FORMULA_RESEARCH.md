# 📊 라테일 전투력 공식 연구 보고서

## 1. 연구 배경

라테일은 전투력 공식을 공식적으로 공개하지 않으므로, **15개의 실제 T창 데이터를 기반으로 회귀분석**하여 공식을 역산했습니다.

## 4. 도출된 공식 (V_RICH 모델)

### 4-1. 공식 형태

```
공격력 베이스 = 주스탯 + 공격력×K1 + 고댐×K2 + (최소뎀+최대뎀)×K3

전투력 = 공격력 베이스
       × (1 + 크댐 / D_crit)
       × (1 + (일몬지+보몬지) / D_dom)
       × (1 + 근마효율 × K_근마)
       × base_constant
```

### 4-2. 물리 직업 파라미터 (12개 데이터로 학습, RMSE 1.18%)

```javascript
const PHYSICAL_PARAMS = {
  K1: 142.70,        // 공격력 가중치 (무공/속성력)
  K2: 2.5456,        // 고댐 가중치
  K3: 0.0788,        // (최소뎀+최대뎀) 가중치
  D_crit: 1e-6,      // 크댐 분모 (사실상 0에 수렴 → 크댐이 매우 강하게 곱셈)
  D_dom: 51.84,      // 지배력 분모
  K_geunma: 0.0046,  // 근마효율 계수
  base: 1e-6         // 전체 상수 (D_crit과 함께 매우 작음)
};
```

> ⚠️ **D_crit과 base가 매우 작은 값으로 수렴한 것은 통계적 한계**입니다.
> 실제로는 크댐 항이 곱셈에서 분리되어 base와 합쳐진 형태로 작동합니다.
> 추가 데이터로 보정 필요. **하지만 12개 데이터에서는 RMSE 1.18%로 잘 작동**합니다.

### 4-3. 마법 직업 파라미터 (3개 데이터, 데이터 부족)

마법 직업은 데이터가 부족하여 별도 공식 도출이 어렵습니다.


## 6. 알려진 한계

### 6-1. 효과를 분리하지 못한 스탯
다음 3개 스탯은 데이터 부족으로 정확한 가중치 산출 실패:
- **일몬추** (일반 몬스터 추가 데미지)
- **보몬추** (보스 몬스터 추가 데미지)
- **관통력** (모든 데이터에서 99로 동일하여 분석 불가)

→ 장비 비교 시 이 스탯들의 변화는 정확하게 반영 어려움

### 6-2. 단일 변수 통제 데이터 부재
같은 캐릭터에서 한 스탯만 변화시킨 데이터가 없어, 각 스탯의 순수 기여도 분리 한계

### 6-3. 다중공선성
주스탯과 공격력, 크댐과 지배력이 상관관계 높아 정확한 분리 어려움

### 6-4. 마법 직업 데이터 부족
3개 데이터로는 별도 공식 학습 불가

## 7. 공식 재학습 방법

### 7-1. 데이터 추가 방식

`SAMPLE_DATA.json`에 새 객체를 추가:

```json
{
  "name": "신규데이터1",
  "type": "P",
  "전투력": 4500000,
  "주스탯": 7000000,
  "공격력": 65000,
  "관통": 99,
  "크댐": 9500,
  "최소뎀": 8000,
  "최대뎀": 8200,
  "고댐": 950000,
  "일몬추": 1100000,
  "보몬추": 1050000,
  "일몬지": 65.0,
  "보몬지": 75.0,
  "근마효율": 18
}
```

### 7-2. 자동 재학습 스크립트

`scripts/refit_formula.py`에 다음 코드를 사용 (Python 필요):

```python
import json
import numpy as np
from scipy.optimize import minimize

# 데이터 로드
with open('SAMPLE_DATA.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

# 물리 직업만 필터링
data_p = [d for d in data if d['type'] == 'P']

# 모델 정의
def model(d, p):
    base = d['주스탯'] + d['공격력']*p[0] + d['고댐']*p[1] + (d['최소뎀']+d['최대뎀'])*p[2]
    if base <= 0: return 0
    return base * (1+d['크댐']/p[3]) * (1+(d['일몬지']+d['보몬지'])/p[4]) * (1+d['근마효율']*p[5]) * p[6]

# 손실 함수
def loss(p, dataset):
    if any(x <= 0 for x in p): return 1e10
    err = 0
    for d in dataset:
        pred = model(d, p)
        if pred <= 0: return 1e10
        err += ((pred - d['전투력']) / d['전투력']) ** 2
    return err

# 최적화 (다양한 시작점)
best = float('inf')
bp = None
for s in range(500):
    np.random.seed(s)
    x0 = [np.random.uniform(80, 200), np.random.uniform(0.01, 5), np.random.uniform(0.001, 1),
          np.random.uniform(20, 200), np.random.uniform(30, 300), np.random.uniform(0.001, 0.05),
          np.random.uniform(1e-7, 1e-3)]
    r = minimize(loss, x0, args=(data_p,), method='Nelder-Mead', options={'maxiter':20000})
    if r.fun < best:
        best = r.fun; bp = r.x

print(f'RMSE: {np.sqrt(best/len(data_p))*100:.4f}%')
print(f'PHYSICAL_PARAMS = {{')
print(f'  K1: {bp[0]:.4f},')
print(f'  K2: {bp[1]:.4f},')
print(f'  K3: {bp[2]:.4f},')
print(f'  D_crit: {bp[3]:.4f},')
print(f'  D_dom: {bp[4]:.4f},')
print(f'  K_geunma: {bp[5]:.6f},')
print(f'  base: {bp[6]:.10f}')
print(f'}}')

# 검증 출력
for d in data_p:
    pred = model(d, bp)
    err = (pred - d['전투력']) / d['전투력'] * 100
    print(f"  {d['name']:25s}: 오차 {err:+.2f}%")
```

이 스크립트를 실행하면 새 파라미터가 콘솔에 출력됩니다.
출력된 값을 `src/utils/battlePower.js`의 `PHYSICAL_PARAMS`에 그대로 붙여넣으면 됩니다.

### 8-1. 가장 가치 있는 데이터 (우선순위 높음)

   - 가장 강력함. 이런 데이터 5개만 있어도 공식이 비약적으로 정확해짐

### 8-2. 데이터 수집 체크리스트

T창 캡처 시 다음 모두 보이게:
- ✅ 전투력 (능력치 세부정보 상단)
- ✅ 근력/마법력 둘 다
- ✅ 무기공격력 (min~max) / 속성력
- ✅ 관통력
- ✅ 크리티컬 데미지 (물리/마법)
- ✅ 최소/최대 데미지 (물리/마법)
- ✅ 고정 데미지 증가
- ✅ 몬스터 추가 데미지 (일반/보스)
- ✅ 몬스터 지배력 (일반/보스, %)
- ✅ 추가 세부정보의 "근력/마법력 효율" %

### 8-3. 도핑/버프 상태 표기
