# 🎮 라테일 전투력 비교 시뮬레이터 - 프로젝트 개요

## 1. 프로젝트 목적

라테일(LaTale) 게임에는 **장비 교체 시 전투력 변화를 미리 보여주는 기능이 없어서**
유저들이 경매장에서 장비 구매 시 큰 불편을 겪고 있음. (메이플스토리 등 다른 게임은 제공)

이 사이트는:
1. 사용자가 본인의 T창(능력치 세부정보) 정보를 입력
2. 현재 끼고 있는 장비 옵션을 입력
3. 비교하려는 새 장비의 옵션을 입력
4. **전투력이 얼마나 변화하는지(상승/하락) 즉시 출력**

## 2. 핵심 가치 제안

- **유일한 라테일 전투력 시뮬레이터** (현재 시장에 없음)
- 경매장 구매 의사결정 직접 도움 → 유저들이 실제로 사용할 동기 강함
- 광고 수익 가능성 (커뮤니티에서 입소문 타기 좋은 유틸)

## 3. 기술 스택 (확정)

| 영역 | 선택 |
|------|------|
| 프론트엔드 | **Vue 3 + Composition API** |
| 빌드 도구 | **Vite** |
| 스타일 | **Tailwind CSS** |
| 상태 관리 | Pinia (필요 시) |
| 데이터 저장 | **LocalStorage** (Phase 1) |
| 언어 | JavaScript (TypeScript도 무방) |
| UI 언어 | **한국어** |
| 호스팅 | Vercel 또는 Netlify (무료) |
| 백엔드/DB | **Phase 1에서는 없음** |

## 4. 단계별 로드맵

### Phase 1 (MVP) ← 지금 만드는 것
- Vue 단독, LocalStorage만 사용
- 핵심 기능: 전투력 계산기 + 장비 비교
- 기능 명세는 아래 "Phase 1 MVP 기능 명세" 참고

### Phase 2 (데이터 수집 시작)
- 익명 데이터 수집 (Google Apps Script + 구글시트로 백엔드 없이)
- 사용자가 자기 T창 정보 입력 시 옵트인으로 익명 전송
- 모인 데이터로 공식 자동 보정

### Phase 3 (트래픽 확장)
- Google AdSense 광고 부착
- SEO 최적화
- (필요 시) 본격 백엔드 + DB 추가

## 5. Phase 1 MVP 기능 명세

### 5-1. 입력 영역

**캐릭터 기본 정보 (T창 - 능력치 세부정보)**
- 직업 타입: 물리 / 마법 (라디오 버튼)
- 전투력 (현재 표시값)
- 주스탯: 근력 또는 마법력
- 공격력: 무기공격력 또는 속성력 (min~max 평균값 사용)
- 관통력
- 크리티컬 데미지
- 최소 데미지
- 최대 데미지
- 고정 데미지 증가
- 일반 몬스터 추가 데미지
- 보스 몬스터 추가 데미지
- 일반 몬스터 지배력 (%)
- 보스 몬스터 지배력 (%)
- 근력/마법력 효율 (%)

→ 마법 직업 선택 시 라벨이 자동으로 "마법력", "속성력", "마법 크댐" 등으로 바뀌게

### 5-2. 장비 비교 영역

**현재 장비 옵션** (이 장비를 빼면 차감되는 스탯)
- 부위 선택: 무기/투구/상의/하의/장갑/신발/액세서리 등
- 옵션 입력: 위 13개 스탯 중 해당하는 것

**비교할 장비 옵션** (이 장비를 끼면 추가되는 스탯)
- 동일한 입력 필드

### 5-3. 결과 출력 영역

```
🟢 전투력 +18,420 상승

기존 전투력: 250,000
교체 후 예상: 268,420

세부 변화:
  - 근력 +200 → 전투력 +4,000
  - 무기공격력 -200 → 전투력 -3,500
  - 크댐 +5 → 전투력 +12,000
  - 최대뎀 -30 → 전투력 -1,580
  - 추가뎀(보스) +5% → 전투력 +7,500
```

### 5-4. 캐릭터 저장 (LocalStorage)

- 입력한 T창 정보를 캐릭터로 저장
- 여러 캐릭터 관리 가능
- 캐릭터 선택 시 자동으로 입력 폼에 채워짐

### 5-5. 비교 히스토리

- 최근 10개 비교 결과 LocalStorage에 저장
- 히스토리 클릭 시 결과 다시 보기 가능

### 5-6. 익명 데이터 기여 (옵트인)

- "내 T창 정보를 익명으로 기여하기" 체크박스
- 체크 시 Google Apps Script webhook으로 데이터 전송
- 전송 데이터: 13개 스탯 + 직업타입 + 전투력 (개인정보 없음)
- ⚠️ Phase 2에서 활성화. Phase 1에서는 UI만 만들고 비활성화 상태로 둬도 됨

### 5-7. 다크모드

- 라이트/다크 토글 버튼
- 시스템 설정 자동 감지 (prefers-color-scheme)

### 5-8. 모바일 반응형

- 모바일에서도 동작 (Tailwind의 sm/md/lg 브레이크포인트 사용)
- 라테일 유저들이 게임 중 핸드폰으로 빠르게 계산하는 시나리오 고려

## 6. 정확도 및 한계 표시

사이트 어딘가에 명시:

> ⚠️ **베타 버전입니다**
> - 물리 직업 정확도: 약 95~98%
> - 마법 직업 정확도: 약 85~95% (데이터 추가 수집 중)
> - 일부 스탯(추가 데미지, 관통력)의 효과는 추정치이며 실제와 다소 차이날 수 있습니다.
> - 데이터가 누적될수록 정확도가 개선됩니다.

## 8. 알려진 제약사항

1. **데이터 부족**: 15개 데이터로 도출한 공식이라 정확도 한계 있음
2. **마법 직업 데이터 부족**: 3개뿐이라 별도 공식 도출 필요
3. **일부 스탯 효과 분리 불가**: 일몬추/보몬추/관통의 정확한 계수 미확인
4. **단일 캐릭터 데이터로 검증 안 됨**: 같은 캐릭터에서 장비만 바꾼 데이터가 없음

→ 이런 한계는 사이트에 명시하고, 사용자가 데이터를 기여하면 점진적으로 개선됨

## 9. 향후 데이터 보완 시 작업

집에서 추가 데이터를 모으면 다음 흐름으로 보완:

1. `SAMPLE_DATA.json`에 새 데이터 항목 추가
2. `scripts/refit_formula.py` 실행 → 새 파라미터 도출
3. `src/utils/battlePower.js`의 파라미터 상수 업데이트
4. `tests/formula.test.js`로 정확도 재검증
5. 정확도 향상 확인 후 배포

또는 Claude Code에게 "데이터 추가했으니 공식 재학습해줘"라고 부탁해도 OK.

## 10. 향후 광고/수익화 시 주의사항
